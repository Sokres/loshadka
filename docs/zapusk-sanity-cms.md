# Как запустить CMS (Sanity) для сайта «Лошадка»

CMS у этого проекта — **[Sanity](https://www.sanity.io/)**. Редактор встроен в Next.js и открывается по адресу **`/studio`** (не отдельный сайт на порту 3333, если вы не запускаете Sanity CLI отдельно).

---

## Что нужно один раз

1. **Аккаунт Sanity**  
   Зарегистрируйтесь на [sanity.io](https://www.sanity.io/) (можно через GitHub/Google).

2. **Проект и dataset**  
   - Зайдите в [sanity.io/manage](https://www.sanity.io/manage).  
   - Создайте проект (или используйте существующий).  
   - Скопируйте **Project ID** из настроек проекта.  
   - Dataset по умолчанию часто называется **`production`** — его можно оставить.

3. **Переменные окружения в репозитории**  
   В корне проекта создайте файл **`.env.local`** (он не должен попадать в git). Скопируйте из `.env.example` и заполните:

   ```env
   NEXT_PUBLIC_SANITY_PROJECT_ID=ваш_project_id
   NEXT_PUBLIC_SANITY_DATASET=production
   NEXT_PUBLIC_SITE_URL=http://localhost:3000
   ```

   Без **`NEXT_PUBLIC_SANITY_PROJECT_ID`** сайт всё равно покажет контент из **fallback** в коде, но **Studio не заработает нормально** (нужен реальный ID проекта).

4. **Custom studio URL в [sanity.io/manage](https://www.sanity.io/manage)** (если Sanity предлагает добавить «Studios you are hosting manually»):  
   укажите адрес **именно редактора**, не главную страницу сайта:
   - локально: **`http://localhost:3000/studio`**
   - продакшен: **`https://ваш-домен/studio`** (например `https://kdkam.ru/studio` при деплое этого приложения)  
   Корень **`http://localhost:3000`** без **`/studio`** здесь не подходит — Studio в этом проекте живёт по пути **`/studio`** (`sanity.config.ts` → `basePath`).

---

## Как открыть редактор (CMS) каждый день

1. Установите зависимости (если ещё не ставили):

   ```bash
   npm install
   ```

2. Запустите приложение:

   ```bash
   npm run dev
   ```

3. В браузере откройте:

   **`http://localhost:3000/studio`**

4. Войдите под тем же аккаунтом Sanity, которому принадлежит проект с этим **Project ID**.

После входа вы увидите деск с типами документов из схем в папке `sanity/schemaTypes/`.

---

## Что создавать в CMS в первую очередь

| Тип документа        | Зачем |
|---------------------|--------|
| **Настройки сайта** | Телефон, WhatsApp, адрес, тексты по умолчанию для SEO |
| **Категория услуг** | Например: прогулки, сезонные, обучение (slug должен совпадать с URL: `progulki`, `sezonnye`, `obuchenie`) |
| **Услуга**          | Маршруты и страницы; для корня сайта включите «Страница на корне» (`standaloneRoute`) для кареты, саней и т.п. |
| **Отзыв**           | Блок отзывов на главной |

Пока документов в Sanity мало или их нет, фронт может подставлять данные из **`lib/cms/types-and-fallback.ts`** (резервный контент в коде).

---

## Полезные детали

- **Vision** в меню Studio — песочница для запросов **GROQ** к вашему dataset (удобно для отладки).  
- **Продакшен-сайт**: переменные `NEXT_PUBLIC_SANITY_*` нужно задать на хостинге (Vercel и т.д.), иначе снова будет только fallback.  
- Конфиг Studio: `sanity.config.ts`, точка входа в приложении: `app/studio/[[...tool]]/page.tsx`.

---

## Отдельный Sanity CLI (по желанию)

Пакет `sanity` уже в **devDependencies**. Если понадобится классическая команда в терминале (деплой хостинга Studio на sanity.io и т.д.), используйте документацию Sanity; для **ежедневной работы** этого сайта достаточно **`npm run dev`** и **`/studio`**.
